================================================================================
PROJECT: Memory Battle Arena (Dual Mode Gamified Learning App)
DOCUMENT: REQUIREMENTS SPECIFICATION
================================================================================

PROJECT OVERVIEW:
A production-ready, responsive, visually engaging memory game supporting two
distinct modes: Fun Mode (visual matching with emojis) and Learning Mode 
(subject-based Q&A matching). The app combines entertainment with education.

================================================================================
FUNCTIONAL REQUIREMENTS
================================================================================

FR-1: MODE SELECTION
  FR-1.1: Display two distinct mode options on app launch
          - Fun Mode (visual emoji matching)
          - Learning Mode (subject-based Q&A matching)
  FR-1.2: Allow user to switch between modes at any time
  FR-1.3: Persist mode preference in session storage
  FR-1.4: Display mode description and preview

FR-2: FUN MODE - BASIC GAMEPLAY
  FR-2.1: Display 4x4 grid of tiles (16 tiles default)
  FR-2.2: Show tile backs when game starts (hidden content)
  FR-2.3: Allow clicking/tapping tiles to flip them
  FR-2.4: Reveal emoji content on tile flip
  FR-2.5: Support multiple themes (animals, food, nature, etc.)
  FR-2.6: Allow difficulty selection (easy, medium, hard)
  FR-2.7: Generate matching pairs based on selected theme/difficulty

FR-3: FUN MODE - MATCHING LOGIC
  FR-3.1: Allow maximum 2 tiles flipped simultaneously
  FR-3.2: Detect matching pairs (tile1.value === tile2.value)
  FR-3.3: Keep matched tiles visible permanently
  FR-3.4: Flip non-matching tiles back after 1.5 second delay
  FR-3.5: Prevent interaction during flip-back animation
  FR-3.6: Prevent clicking same tile twice

FR-4: LEARNING MODE - SETUP
  FR-4.1: Display subject selection dropdown (Math, Science, History, etc.)
  FR-4.2: Display topic selection based on chosen subject
  FR-4.3: Display difficulty selection (easy, medium, hard)
  FR-4.4: Generate Q&A tile pairs from selected subject/topic/difficulty
  FR-4.5: Display question on one tile, answer on paired tile
  FR-4.6: Assign pairId to link questions and answers

FR-5: LEARNING MODE - MATCHING LOGIC
  FR-5.1: Detect matching pairs (tile1.pairId === tile2.pairId)
  FR-5.2: Show "Correct!" message with explanation on match
  FR-5.3: Show "Try again" message on non-match
  FR-5.4: Display learning explanation from JSON data
  FR-5.5: Track correct and incorrect matches separately

FR-6: GAME FEATURES - CORE
  FR-6.1: Display move counter (increments each pair attempt)
  FR-6.2: Display timer (starts on first tile click)
  FR-6.3: Display timer in MM:SS format
  FR-6.4: Detect game completion (all pairs matched)
  FR-6.5: Display completion screen with stats
  FR-6.6: Provide restart button to reset game state

FR-7: GAME FEATURES - LEARNING MODE ONLY
  FR-7.1: Provide hint button (limit to 2-3 uses per game)
  FR-7.2: Show revision screen displaying all Q&A pairs
  FR-7.3: Allow retry of only incorrect pairs
  FR-7.4: Display accuracy percentage at completion
  FR-7.5: Track learning progress

FR-8: ANIMATIONS
  FR-8.1: Implement 3D card flip animation on tile click
  FR-8.2: Add glow/bounce effect on successful match
  FR-8.3: Add shake animation on failed match
  FR-8.4: Display confetti animation on game completion
  FR-8.5: Smooth all transitions (200-500ms)
  FR-8.6: Prevent animation clipping/jumping

FR-9: AUDIO SYSTEM
  FR-9.1: Play flip sound on tile click
  FR-9.2: Play match success sound on correct pair
  FR-9.3: Play fail sound on incorrect pair
  FR-9.4: Play win/completion sound
  FR-9.5: Provide mute/unmute toggle button
  FR-9.6: Remember mute preference in localStorage

FR-10: RESPONSIVE DESIGN
  FR-10.1: Support desktop (1024px+), tablet (768px-1024px), mobile (< 768px)
  FR-10.2: Mobile-first responsive grid layout
  FR-10.3: Dynamic tile sizing based on viewport
  FR-10.4: Large tap areas on mobile (minimum 44x44px)
  FR-10.5: No content overflow on any device
  FR-10.6: Readable fonts on all screen sizes

FR-11: DATA MANAGEMENT
  FR-11.1: Load fun.json with theme-based emoji data
  FR-11.2: Load learning.json with subject/topic/Q&A data
  FR-11.3: Handle missing or corrupted JSON gracefully
  FR-11.4: Shuffle tiles on each game start
  FR-11.5: Support dynamic data loading via API simulation

FR-12: ERROR HANDLING
  FR-12.1: Handle missing JSON files
  FR-12.2: Validate data structure before game start
  FR-12.3: Display user-friendly error messages
  FR-12.4: Provide recovery options (reload, main menu)
  FR-12.5: Prevent game crashes from invalid data

================================================================================
CONSTRAINTS
================================================================================

C-1: Technology Stack
     - HTML5, CSS3, Vanilla JavaScript
     - No external libraries (no jQuery, Bootstrap, etc.)
     - No backend services
     - JSON files for data only

C-2: File Structure
     - Modular folder organization required
     - Separated concerns (components, services, utilities, data, styles)
     - Single entry point (index.html)

C-3: Data Management
     - No database required
     - JSON files stored locally
     - Mock API for data fetching

C-4: Browser Environment
     - Client-side only
     - No Node.js or build tools
     - Works as static files
